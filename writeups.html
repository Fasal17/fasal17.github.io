IDOR in Vehicle Booking System

Severity: High

Broken Object Level Authorization allowed access to other users’ data.

Changing the vehicle ID parameter exposed unauthorized booking information.

Admin Panel Exposure

Severity: Critical

Admin login page publicly accessible without IP restrictions.

Attackers could brute-force admin credentials leading to full compromise.

Reflected XSS in Search Endpoint

Severity: Medium

User input was reflected without proper encoding.

The search parameter was directly reflected into HTML response, allowing JavaScript execution. Could lead to session hijacking.

Fix: Output encoding + CSP.

Account Enumeration via Password Reset

Severity: Low

Different responses for valid and invalid emails.

Attackers could enumerate registered users.

Fix: Generic responses + rate limiting.

Improper Rate Limiting on Login

Severity: High

No lockout or CAPTCHA after failed attempts.

Brute-force attacks possible on authentication endpoint.

Fix: Rate limiting + account lockout.

Sensitive Information Disclosure

Severity: Medium

Stack traces revealed internal paths.

Error messages exposed server paths and framework versions.

Fix: Disable debug mode in production.
